Java历史与特性

Java历史发展

　　任何一门新语言的诞生都有其必要性，或是现有的语言不能满足其要求，亦或是要实现我们的需求，使用旧语言会显得啰嗦和容易犯错。总的来说，就是旧语言已经不能再适应我们的需求，那么一门新语言就显得尤为重要。
　　在编程语言还在被C和C++占领的时候，显然说明这两种语言能满足当时的需要，随着时代的发展，在20世纪90年代，单片机计算机开始兴起，由于其具备价格低廉，功能强大的特点，开始被大量的人使用，智能化时代初具雏形。此时SUN公司为了 赢得市场，成立了一个叫Green的项目小组，詹姆斯•高斯林（James Gosling）担任组长，计划用当时流行的C++来编写程序。但是后来因为种种原因，项目组决定对C++进行改造，从而形成了一种新语言，命名为Oak，是一种纯面向对象的语言，这也是Java的前身。
　　比较有趣的是，原本为了嵌入式系统、单片机而研发的这种新语言，在推广的时候，却并不被硬件厂商看好，反而是在后来的互联网时代大放异彩。在注册商标的时候，因为Oak还被注册过了，不得已换了新名字，传闻是他们在喝Java（爪哇）咖啡的时候，随意起的。Oak这个名字也是因为公司门口有棵橡树而得名。不得不感慨，这些程序员在为语言起名是如此的随性。
　　在1996年1月，SUN公司发布了Java1.0。之后就是百无聊赖的版本更迭。其中有两次比较重要的版本，一个是Java1.2，标志着java的应用开始普及；另一个是Java1.5，因版本更新太多，特将版本号改为Java5。在2011年，SUN公司被Oracle公司收购，更新还在继续，到目前为止已经更新到Java14。

Java特性

　　Java是一个纯面向对象的语言，一切东西都应该在对象里面，对象外面不能写任何逻辑代码。至于为什么是逻辑代码，是因为我们的导包代码还是要在对象之外写的，比较有意思。
　　在Java1.2出来的时候，SUN公司将JAva分为了三大板块，分别是JavaSE、JavaEE和JavaME。其中JavaSE是基础，用于桌面端开发；JavaEE则是用于服务器端的开发，是目前的主流；JavaME主要用于消费性电子产品上。
　　Java最核心的特点就是Java虚拟机和垃圾回收机制，Java虚拟机（JVM）让Java程序能够跨平台运行，垃圾回收机制（GC）让程序员能够更加专注于编程逻辑，写出优质代码，而不至于被内存释放这种问题占据思维。这两种特殊机制都是在C和C++上改进得来的。

JVM（Java虚拟机）

　　定义：Java虚拟机是一中运行在操作系统上的虚拟计算机系统，具有将java字节码解释执行的能力。
　　那为什么JVM这么重要呢？那是因为在以前，C语言流行的时候，由编译器编译后的C语言代码可以直接在操作系统运行，但是会出现，不同的操作系统有不同的编译器，编译出来的机器码各不相同，所以将C语言编译后，是不能跨平台运行的。而Java提供了java虚拟机，Java程序运行过程是先将java源代码经过编译，成为字节码文件，然后由虚拟机解释执行字节码文件，这就相当于加了一个中转站，屏蔽了操作系统的差异，我们只要在操作系统上安装对应的java虚拟机就行了。
　　灵魂比喻：我将java虚拟机比喻成一个恒温箱，放在里面的东西总是恒定的，不需要改变的，假如外界条件发生变化，我们也只是换一种对应的恒温箱就可以了，遇到冬天就换抗寒的恒温箱，夏天就换夏天的恒温箱，里面的东西可以到各个地方去，这，就是跨平台。
　　当然，有些老师讲的翻译官的例子也是非常生动形象的，都是要靠自己的理解，希望你也能有属于自己的比喻。

GC（垃圾回收机制）

　　垃圾回收机制也是java的一大重要特性。缘由是因为在C和C++语言中，程序员自己申请的内存，如果发现后面已经不能或者不需要使用的话，应该自己释放内存。但是由于C和C++程序运行在复杂环境下，很多时候连程序员自己也不能确定某一块内存是否应该释放了，就会导致出现一些严重的错误。基于这个原因，java的设计者就想让程序自己来释放内存，以减轻程序员的压力。
　　垃圾回收机制的原理是，在运行程序时，会同步开启一个系统线程用来检测内存，一旦发现有可以回收的内存，就会将内存回收掉。当然，实际的运行过程很复杂，可以以后讨论。反正，得出的结果就是程序员不在需要考虑释放内存的问题了，这个东西已经解决了。

类、方法及注释的讲解

　　类
　　由于Java是纯面向对象的语言，所以在学习之前，先看看类、方法！所谓类，就是本意，即，类型、类别、种类等等。初期我们就只要了解除了导包，其他所有的代码都是写在类里面的就行了。来看看类的定义。public class HelloWorld {}这就定义好了一个空的类。其中public是用来修饰类的访问权限的，这个我们现在只需要了解就行，不需要改动。calss是表示类的关键字，也就是说，看到这个关键字，就表示我们定义的是一个类，而不是其他什么东西。HelloWorld表示的是类名，这个类名就是由我们自己写的，当然也不是随便什么字符都能往里写，是有规定的。
　　类名的规定：强制规定：1.类名中只能存在字母（包括大写和小写字母）、数字、下划线_和美元符$。2.数字不能放在首位。错误示范：2Hello，在代码中写出这种类名的，要打死。
非强制规定 也叫命名规范：一般采用驼峰命名法，就是将类名中的所有单词的首字母大写。示例：HelloWorld其中H和W都是大写的。如果不按这种规范来的话，希望你的代码只有你一个人看。
　　方法
　　方法是定义在类里面的东西，啥也不说，先来看看定义一个main方法吧！public static void main(String[] args){}这就定义好了一个空的main方法。其中public还是修饰方法的访问权限的，static表示该方法是一个静态方法，void表示该方法的没有返回值，main表示方法名，括号中的String[] args表示方法的参数。当然，定义mian方法的话，我上面讲的看看就好，所有东西都不能改，原样照抄就好。（我不会告诉你参数名其实是可以改的）
　　注释
　　注释这个东西我觉得一开始就要知道，并且也应该写，不然你的代码稍微长一点，别人看到你的就会说你的代码是一坨屎。哈哈，开个玩笑！注释又分为三种，单行注释，多行注释和文档注释。单行注释是这样的：//这是我的单行注释，解释一些简单的逻辑。多行注释又是这样的：/*这是我的多行注释，用来解释比较复杂的逻辑*/。文档注释是这样的：/**这里写文档注释，具体好像是能生成一个注释文档（html格式的）*/。不管了先靠单行注释和多行注释撑着吧，等需要文档注释的时候再详细研究文档注释啦！

Java运算符

基本运算符

　　+-*/%
　　这五个运算符和我们数学中的运算符基本一致，要注意的是％这个符号，他表示的是取余，也是一个双目运算符，结果为左边的数字除以右边的数字后的余数。
　　+=　-=　*=　/=　%=
　　这五个运算符则是一种省略的写法，表示的意义是在自身的基础上加、减、乘、除、取余右边的数。示例：a +=b；表示的意义是a在自身的基础上加了b。其余同理。

逻辑运算符

　　& | ^　&&　||
　　这五种运算符虽然也是双目运算符，但是它们运算的不在是单纯的数字了，而是逻辑真和逻辑假。
　　先讲讲&、|、^这三种逻辑运算符，它们分别叫做逻辑与、逻辑或、逻辑异或。
&：需要它左右两边都为逻辑真，整个表达式才为真。
|：需要它左右两边都为逻辑假，整个表达式才为假。
^：需要它两边的逻辑不同，即一个逻辑真一个逻辑假时，表达式才为真。

逻辑与　& 逻辑或　| 逻辑异或　^
两边都真才为真 两边都假才为假 两边不同才为真
短路与　&& 短路或　|
如果左边为假，那么就不判断右边的真假了 如果左边为真，就不判断右边的真假了

一般情况下我们都使用短路与、短路或，因为可以节约性能。只有需要对数字进行逻辑运算时采用逻辑与、逻辑或、逻辑异或。

==与equals的区别与联系

　　在之前我们刚开始学习时，很多时候都是比较基本数据类型是否相等，用的也都是==。==比较的是栈区的值是否相等，对于基本数据类型，数据都是储存在栈区，因此可以直接用==比较。假如用==比较两个字符串是否相等，那么比较的也是栈区的值是否相等，但是对于引用类型来说，栈区储存的是地址，所以比较的是地址是否相等，如果值相等，但是是不同的变量，也就是地址不同的话，比较的结果也是false。
　　而我们想要比较两个引用类型变量的值是否相等时，就需要用到equals()。queals()是比较引用数据类型中真正储存的数据是否相等的一种方法。

变量作用域问题

　　我们一开始编程的时候都会创建创建几个变量打印出来玩玩，一般都是直接在main方法里面直接定义和使用的，中间没有其他多余的语句，这没什么问题。随着写的代码越来越多，我们发现，有时候我们会无意间把变量的定义放在for循环或者if判断里面，然后出了这些语句我们还想用这些变量的时候发现变量找不到了，给出的错误是变量没有被定义。怎么回事？明明已经定义了啊，怎么会说没有定义呢？这就要来说一说变量作用域的问题了。首先我们要明白的是我们定义的所有变量都是有作用域的，超过作用域就无法使用了。但是有些作用域的范围很大，有些作用域很小。我们在一个方法中定义的一个变量，在方法之外我们就无法访问了，这个变量我们也叫做局部变量。在类中的方法之外定义的变量我们叫做成员变量，作用域则是整个类。一般来说最简单直观的分辨方式就是，看变量定义的位置，距离变量定义外部的最近的花括号{}，也就是说看他被哪个花括号包围，那么那整个花括号的范围就是他的作用域。

流程控制

　　很多时候我们并不需要程序的每一步都执行，而是需要程序根据一些条件来判断哪些应该执行，哪些应该跳过。这时候就需要流程控制语句了。一般有几种，if、if-else、if-else if-else if、if-else if-else if-else。其中单独的if语句，就是直接if(条件){条件成立执行的代码}，表示如果我需要程序满足一定条件的时候才执行一段代码，如果不满足就相当于什么也没发生，那么这段代码就可以放在一个单独的if语句里面。而if-else语句是这样写的：if(条件){条件成立应该执行的代码}else{条件不成立应该执行的代码}，表示我需要程序满足一个条件就执行一段代码，但是条件不成立也不能相当于什么也没发生，也应该执行一段代码，给我另一种反馈。后面的两种就比较好理解了，是在前面两个的基础上的，意思是如果我有多个条件，而且是后一个条件是在前面那个条件不满足的情况才会判断，要是第一个条件就满足了，那也就没有后面条件的事了，翻译成中文就是，如果···就···不然···再···再不然···再···。最后两个的区别在于，没有加最后的else的，这么多条件，可能都不符合，也就都不会执行，但是有else的，即便你提的所有条件都不满足，也会执行else后面的代码。

几种循环的使用

　　有的时候我们需要重复执行一些相同的代码，当然可以写很多一样的代码，但是这样并不是很好，如果想要修改重复执行的代码会很麻烦，浪费很多时间和精力。于是循环语句出现了，它可以允许我们简单使用几行代码就可以实现重复执行的效果，并且修改要循环的语句也相当容易，只需修改一次就可以了。java提供了三种循环，分别是for循环，while循环和do while循环。我们可以根据情况的不同来选择不同的循环语句。for循环一般用来循环那些需要指定循环次数的代码，或者遍历数组。while循环更适合需要特定条件来结束循环的代码，需要你自己来更新条件。而do while循环与while循环很类似，区别就是do while循环是先开始执行一次代码，才开始判断，哪怕你写的条件判断刚开始就不满足条件，那也会执行一次。

break与continue

　　在执行循环的时候，我们可能有更加苛刻的条件，比如说，我们中间有一条循环不想执行的话，或者在执行到某一个循环时直接退出，不执行了。面对这样的要求，显然上面的循环不能做到，需要借助其他代码，就是break和continue。在循环的代码块中，执行到continue语句时，会直接跳过本次循环进行下一次循环。执行了break的话，就会直接跳出本层循环，循环结束。注意，单独的continue和break都只是针对当前循环而言的，代码中如果有循环嵌套的话，就需要多次跳出。如果真是有需求要跳出指定的循环，可以给循环起个别名，具体做法是在循环前面加别名并加上冒号，然后使用break 别名;来跳出指定的循环，或者使用continue 别名;来进行指定循环的下一次循环。

几种权限修饰符

　　一共有四种权限修饰符，按权限从大到小分别是public（公有的）、protected（受保护的）、default（默认的）、private（私有的）。用的最多的是public和private,如果不写权限修饰符的话，就是default默认权限修饰。下面使用一个表格来展示这四个权限修饰符的访问权限。

public protected default private
当前类 ✔ ✔ ✔ ✔
当前包的其他类 ✔ ✔ ✔ ✘
其他包的当前类的子类 ✔ ✔ ✘ ✘
其他包的其他类 ✔ ✘ ✘ ✘

数组的冒泡排序和二分算法

　　当我们想要对一个整数数组里的值进行排序时，有很多种方法，但是比较常见的一种是冒泡排序。思路是：将数组内的相邻数字进行比较，若左边的数字大于右边的数字则将他们的位置互换，若左边小于右边，则位置不变。经过一轮比较，我们会将最大的一个数字移到数组的最右边，那么在进行n-1轮的比较之后，我们就可以得到一个有序的数组（从左至右数字越来越大）。
　　二分算法是一个用来查找数据的算法，用于在一个有序数组中寻找一个数，效率比直接遍历数组查找要高。思路是：首先定义三个变量分别存储最大值位置，最小值位置和中间值位置，一开始最小值自然是数组的第一个值，最大值是数组的最后一个值，中间值位置等于最大值位置加上最小值位置再除以2（建议：当数组长度非常大的时候，需要考虑一件事，就是最大值的位置加上最小值的位置可能会超过int的最大值，为了防止溢出，我们可以采用min+(max-min)/2，这样即便计算出的最大值的位置和最小孩子的位置都非常靠后，也不用担心会发生数字溢出的错误了）。其次，将我们要查找的数值与中间为之的值比较，如果查询的数字大于中间位置的数字，那么就将最小值位置移动到中间位置加1，如果查询的数字小于中间位置的数字，那么就将最大值的位置移动到中间值位置减1。执行完这个操作后都将中间值的位置更新。这个循环退出的条件是，当发现最小值的位置比最大值的位置还大，如果这时还没找到我们想找的数字，那么就打印说没有找到你想找的数字。

第三章 面向对象编程

面向对象编程思维

　　面向对象编程是一种新的编程思维。在过去用的比较多的是面向过程编程的思维。我们目前学习的面向对象编程，是将一系列相关的数据和方法集合起来放在类里面，我们需要完成什么事情可以创建对象，然后调用对象中的方法来完成一系列的事情。

面向对象的三大特征

　　面向对象的三大特征是封装、继承和多态。
封装：就是将类里面的属性都私有化，让对象都成为独立的，不会受到外界的直接干扰，能够有效的保证数据的安全性。
继承：当需要在一个类的基础上新建一个类的时候，就会用到继承，它的关键字是extends。需要注意的是，虽然子类继承了父类的一切，但是对于父类的私有属性，子类还是不能直接访问，只能通过getter和setter方法来访问。
多态：当我们在继承的时候，可以添加自己特有的属性和方法，继承同一个父类，每个子类都可以是不同的，在创建对象的时候，左边的类型可以是父类，而右边可以通过子类的构造方法来创建对象，我们无法判断右边是通过什么构造方法来构造对象的，这就是多态。

this关键字和static关键字

　　this关键字代表的就是本类或者本对象。一般用在构造方法和setter方法里面。可以将参数赋值给本类或者本对象的属性。
　　static关键字指的是静态的，用static修饰的变量和方法代表的是类的属性和方法，一个类只有一个，无论创建出多少对象，都只会有一个，所有对象都同用这一个。

代码块

　　代码块分为普通代码块、构造代码块和静态代码块。普通代码块就是我们经常使用的，包括if代码块for循环和while循环代码块等等。
构造代码块：就是写在类里面的一个花括号{}，前后不加任何修饰，花括号之间可以写一些代码。这些代码会在对象每次创建的时候运行。一个类里面可以写多个构造代码块，每次执行的时候都会按顺序执行。
静态代码块：就是在构造代码块的基础上，在左花括号前面加上static，表示的就是静态代码块，静态代码块是在类被加载的时候执行的，只会执行一次，无论我们创建多少个对象都只会执行一次。一般在只需要准备一次的数据上使用。
